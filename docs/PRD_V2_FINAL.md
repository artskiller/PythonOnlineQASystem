## 1. 产品概述

### 1.1 产品简介

**pythonLearn** 是一款面向Python初学者的交互式学习平台，专为单机或局域网环境（500人以内）设计。平台通过"做中学"的理念，提供丰富的编程题目、实时代码执行、智能提示系统、学习规划、激励机制等功能，帮助学习者高效掌握Python编程技能。

### 1.2 产品愿景

**让Python学习变得简单、有趣、高效**

- **简单**: 降低学习门槛，提供渐进式学习路径
- **有趣**: 游戏化激励，社区互动，增强学习乐趣
- **高效**: 智能推荐，错题本，个性化学习，提升学习效率

### 1.3 核心价值主张

#### 对学生
- ✅ **即时反馈**: 实时代码执行，立即看到结果
- ✅ **智能提示**: 3级渐进式提示，不直接给答案
- ✅ **个性化学习**: 根据能力推荐题目，定制学习路径
- ✅ **社区互助**: 讨论区、问答社区，遇到问题不孤单
- ✅ **成就激励**: 积分、等级、成就系统，保持学习动力

#### 对教师
- ✅ **班级管理**: 轻松管理多个班级和学生
- ✅ **作业系统**: 一键布置作业，自动批改，节省时间
- ✅ **数据洞察**: 查看学生学习数据，识别薄弱点
- ✅ **教学资源**: 丰富的题库和课程资源

#### 对机构
- ✅ **部署简单**: Docker一键部署，无需复杂配置
- ✅ **成本低廉**: 单机/局域网部署，无需云服务器
- ✅ **数据安全**: 数据本地存储，完全可控
- ✅ **可扩展**: 支持500用户，满足中小型机构需求

### 1.4 产品定位

**目标市场**: 中小型教育机构、学校、培训班、自学小组

**部署方式**: 单机或局域网部署（非云端SaaS）

**用户规模**: 500人以内

**技术栈**:
- 前端: Vue.js 3 + Vite + CodeMirror 6 + Tailwind CSS
- 后端: Python 3.8+ + Flask + SQLAlchemy + Docker SDK
- 数据库: SQLite 3.35+ (支持迁移到MySQL)
- 部署: Docker + Docker Compose + Nginx

## 2. 市场分析

### 2.1 市场机会

#### 编程教育市场增长
- 全球编程教育市场规模持续增长
- Python成为最受欢迎的编程语言之一
- 在线编程学习需求旺盛

#### 现有产品痛点
- **云端平台**: 需要稳定网络，数据安全担忧，成本高
- **传统教学**: 缺少即时反馈，学习效率低
- **开源工具**: 功能分散，缺少系统化学习路径

### 2.2 竞品分析

| 竞品 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|-------------|
| **LeetCode** | 题库丰富，社区活跃 | 面向求职，难度高，需要网络 | 面向初学者，本地部署，教学功能强 |
| **Codecademy** | 交互式学习，课程系统 | 云端SaaS，收费高，英文为主 | 本地部署，免费，中文友好 |
| **实验楼** | 实验环境，项目实战 | 需要网络，资源消耗大 | 轻量级，单机可用 |
| **传统IDE** | 功能强大，专业 | 学习曲线陡，缺少学习引导 | 专为学习设计，智能提示 |

### 2.3 目标市场

1. **培训机构**: Python培训班、编程训练营
2. **高校**: 计算机专业、非计算机专业Python课程
3. **企业内训**: 企业员工Python技能培训
4. **中学**: 信息技术课程、编程兴趣班
5. **自学小组**: 学习小组、编程社团

    output: "nohtyP"
    visible: false

  level1: |
    思考方向：Python字符串切片操作

### 10.3 CI/CD流程

**流程**:
1. 代码提交 → 自动运行单元测试
2. 合并到develop → 自动部署到测试环境
3. 合并到main → 自动部署到生产环境

**工具**: GitHub Actions

**配置示例**:
```yaml
name: CI/CD

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Run unit tests
        run: |
          pytest --cov=. --cov-report=xml
      - name: Check coverage
        run: |
          coverage report --fail-under=80

  deploy:
    needs: test
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to production
        run: |
          docker-compose up -d
```

### 10.4 Bug管理流程

**流程**:
1. 发现Bug → 创建Issue
2. 分类和优先级排序
3. 分配给开发工程师
4. 修复 → Code Review → 测试
5. 关闭Issue

**优先级定义**:
- **P0 - 紧急**: 系统崩溃、数据丢失、安全漏洞（24小时内修复）
- **P1 - 高**: 核心功能不可用（3天内修复）
- **P2 - 中**: 功能异常但有workaround（1周内修复）

## 13. 风险与挑战

### 13.1 技术风险

#### 风险1: Docker容器性能 🔥🔥🔥

**描述**: 大量并发代码执行可能导致性能问题
**影响**: 高（影响核心功能）
**概率**: 中（50%）
**风险值**: 高 × 中 = 高

**缓解措施**:
1. **技术预研**: 在MVP前进行性能测试，验证可行性
2. **容器池管理**: 预创建容器池，减少启动时间
3. **限制并发**: 限制同时执行的容器数量（≤20）
4. **队列机制**: 超过并发限制时进入队列等待
5. **性能监控**: 实时监控容器性能，及时发现问题

**应急方案**:
- 如果性能不足，考虑使用更轻量的沙箱方案（如pypy-sandbox）
- 或者限制代码执行功能的使用频率

---

#### 风险2: SQLite性能瓶颈 🔥🔥

**描述**: 用户数增长可能超出SQLite性能限制
**影响**: 中（影响系统扩展性）
**概率**: 中（50%）
**风险值**: 中 × 中 = 中

**缓解措施**:
1. **性能测试**: 在技术预研阶段验证SQLite性能
2. **优化查询**: 使用索引、优化SQL查询
3. **缓存机制**: 使用Redis缓存热点数据
4. **连接池**: 使用连接池管理数据库连接
5. **迁移方案**: 使用SQLAlchemy ORM，预留MySQL迁移能力

**应急方案**:
- 如果用户数超过500，立即迁移到MySQL
- 迁移成本: 2周开发时间

---

#### 风险3: 代码执行安全 🔥🔥🔥

**描述**: 恶意代码可能突破沙箱限制
**影响**: 高（安全问题）
**概率**: 低（20%）
**风险值**: 高 × 低 = 中

**缓解措施**:
1. **严格的资源限制**: CPU、内存、磁盘、网络
2. **网络隔离**: 禁止访问外网
3. **文件系统隔离**: 只读挂载
4. **危险操作拦截**: 禁止import os, subprocess等
5. **安全测试**: 定期进行安全测试，尝试突破沙箱
6. **及时更新**: 及时更新Docker和Python版本

**应急方案**:
- 如果发现安全漏洞，立即禁用代码执行功能
- 修复后重新上线
